# 选股模块架构与运行流程

## 入口

```bash
python -m 选股.cli scan --pool 沪深主板 --top 30
```

CLI 解析参数后调用 `scanner.py` 的 `scan_all()`，整个流程分为 **五个阶段**。

---

## 第一阶段：股票池构建（`pool.py`）

### 五种可选的股票池

| 池名 | 数据来源 | 大约只数 |
|------|----------|---------|
| `自选` | 读取 `观察仓/watchlist.txt` | 手动维护 |
| `沪深300` | 东方财富 API → `b:MK0146` | ~300 |
| `中证500` | 东方财富 API → `b:MK0021` | ~500 |
| `全A` | 东方财富 API → 4 个市场合并 | ~5000 |
| `沪深主板` | API 优先，失败则降级到本地静态文件 | ~5896（静态） |

### `沪深主板` 的特殊双路径机制

```
get_stock_pool("沪深主板")
  │
  ├── 路径A：尝试 fetch_stock_list("全A") → 调用东方财富 API
  │    成功 → 从全A中过滤掉 300xxx/301xxx/688xxx/8xxxxx → 返回精确名单
  │
  └── 路径B：API 不可达时 → _load_static_main_board()
        文件存在 → 直接从 stock_list_main_board.json 加载
        文件不存在 → _generate_static_main_board() 动态生成
            └── 遍历 8 个预定义的沪深主板代码区间（如 600000-601999、603000-603999 等）
            └── 排除 300/301/688/8 开头的代码
            └── 生成 ~5896 条 code 列表，写入 JSON 文件，24h 内复用
```

### 缓存策略

每个池的股票列表会按池名索引缓存到 `选股/stock_pool.json`，有效期 24 小时。同一池名在 24 小时内不会重复调用 API，直接读缓存。

## 第二阶段：逐票扫描（`scanner.py` → `scan_one()`）

这是核心循环——对股票池中每只股票执行相同的流水线，任一环节失败即跳过该票。

### 步骤 1：数据抓取

```
fetch_kline(code, 350, "day")
```

调用东方财富 K 线 API（`push2his.eastmoney.com`），取 150+200=350 根日 K 线（200 根为指标预热用）。失败则尝试 1 次重试，仍失败则排除该票。

### 步骤 2：基础过滤（两道硬门槛）

- **上市天数检查**：350 根 K 线 > 120 根（否则均线尚未成型）
- **成交量有效性**：近 20 根 K 线中最低成交量 / 平均成交量 > 0.1（排除停牌/一字板）

### 步骤 3：双系统指标计算

**无情操盘手**（`criteria/ruthless.py` → 复用 `backtest_ruthless.py`）：

| 指标 | 计算方式 |
|------|---------|
| 白线 | DEMA(close, 10) = EMA(EMA(close, 10), 10) |
| 黄线 | (MA14 + MA28 + MA57 + MA114) / 4 |
| K/D/J | KDJ(9, 3, 3)，从 K 线高低收计算 RSV |
| 砖柱 | DIF=白-黄，DEA=EMA(DIF,9)，砖=(DIF-DEA)×2，红/黄/绿/蓝四色 |

**波浪理论**（`criteria/wave.py` → 复用 `fetch_wave_data.py`）：

| 指标 | 计算方式 |
|------|---------|
| MACD | MACD(5, 34, 5)，Fast=5，Slow=34，Signal=5 |
| 摆动点 | 左右各 5 根 K 线确认的局部高低点 |
| 背离 | 价格新低 DIF 走高 = 底背离；价格新高 DIF 走低 = 顶背离 |
| Fibonacci | 基于最近一波主升/主跌的高低点计算回撤位 |
| 成交量 | 近 20 天 vs 前期 20 天的均量对比 |

两组指标各算一次，后续全部条件共享这些指标数据，不重复计算。

### 步骤 4：排除过滤（两道信号门槛）

- **`EXCLUDE_DEATH_CROSS`**：白线下穿黄线 → 直接排除
- **`EXCLUDE_BELOW_YELLOW`**：收盘价跌破黄线 → 直接排除

这是无情操盘手的核心纪律——标准战法只在价格高于黄线的强势格局中操作。不满足的票即使其他信号再好也不纳入。

### 步骤 5：逐条件打分（13 条规则）

合并生成统一的 `ind` 字典（含 ruthless + wave 全部指标），然后遍历 `config.py` 中配置的 **13 条筛选条件**，每条条件函数被调用后返回 `(得分, 详情)`。

**无情操盘手 5 条（权重合计 45）：**

| 条件 | 满分 | 逻辑 |
|------|------|------|
| B1 超卖 | 15 | 近 5 根 K 线内 detect_b1 触发 → 满分；仅 J<30 → 半分 |
| B 砖反转 | 10 | 绿→蓝→黄 + B2 阳线确认 → 满分；仅黄砖出现 → 半分 |
| 白黄金叉 | 10 | 近 5 日内白线上穿黄线 → 满分 |
| 强势排列 | 5 | 价格 > 白线 > 黄线 → 满分；夹心 → 半分 |
| KDJ 底金叉 | 5 | J < 30 且 K 上穿 D → 满分；仅 J < 30 → 1/3 |

**波浪理论 5 条（权重合计 40）：**

| 条件 | 满分 | 逻辑 |
|------|------|------|
| MACD 金叉 | 10 | 近 5 日内 DIF 上穿 DEA → 满分；金叉持有中 → 半分 |
| MACD 底背离 | 12 | 近 60 天内有底背离 → 满分；有但较远 → 半分 |
| Fib 支撑 | 8 | 收盘价在 38.2%/50%/61.8% 回撤位的 ±2% 内 → 满分 |
| 缩量止跌 | 5 | 量缩到前期的 70% 以下 + 振幅 < 4% → 各给一部分分 |
| 波浪位有利 | 5 | DIF 近零轴 + Fib 支撑位附近 + 红柱回升 → 启发式评分 |

**加分项 3 条（权重合计 15）：**

| 条件 | 满分 | 逻辑 |
|------|------|------|
| 多重底背离 | 5 | ≥3 个底背离 → 满分；2 个 → 半分 |
| Fib 汇聚 | 5 | 两个不同来源的 Fib 位相距 < 3% 且当前价在其附近 |
| 放量阳线确认 | 5 | 近 3 日内实体 ≥ 2% + 量 > 5 日均量 × 2 |

### 步骤 6：汇总与过滤

- 收集所有条件得分（每条 `score > 0` 才计入），求和得到总分
- 总分 < **`MIN_SCORE`**（默认 25）→ 排除
- 总分 ≥ 25 → 输出完整分析结果，包含逐条件评分明细

---

## 第三阶段：排序与截断（`scanner.py` → `scan_all()`）

- 所有通过 `scan_one()` 的股票按 **总分降序** 排列
- 取前 **`TOP_N`**（默认 30）名
- 整个过程打印逐批进度（每 50 只显示一次扫描速度和 ETA）

## 数据流全景图

```
CLI: python -m 选股.cli scan --pool 沪深主板 --top 30
  │
  ▼
[第一阶段] pool.py::get_stock_pool("沪深主板")
  ├─ API可达 → 东方财富全A列表 → 过滤主板 → ~3300 只
  └─ API不可达 → 静态代码区间 → ~5896 只
  │
  ▼
[第一阶段] pool.py::filter_stocks() → 去ST、新股
  │
  ▼
[第二阶段] scanner.py::scan_all()
  │
  for each (code, name):
    │
    ▼
    scan_one(code, name)
    ├─ [2.1] fetch_kline(code, 350, "day") ─── 东方财富 K 线 API
    ├─ [2.2] 上市天数 < 120? vol 无效? → 跳过
    ├─ [2.3] build_ruthless_indicators() ──── 白线/黄线/KDJ/砖柱
    │        build_wave_indicators() ──────── MACD/摆动点/背离/Fib/成交量
    ├─ [2.4] EXCLUDE: 死叉? 跌破黄线? → 跳过
    ├─ [2.5] 13 条 criteria 逐条打分 → sum → total_score
    ├─ [2.6] total_score < MIN_SCORE(25)? → 跳过
    └─ [2.7] 返回 {name, score, details, latest_info, indicators}
  │
  ▼
[第三阶段] sort by score desc → [:TOP_N]
  │
  ▼
[第四阶段] report.py
  ├─ print_summary() → 终端表格
  ├─ generate_markdown() → 结构化报告
  └─ save_report() → 选股/选股结果/YYYY-MM-DD_沪深主板.md
```

## 关键可调参数（`config.py`）

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `EXCLUDE_DEATH_CROSS` | True | 排除白黄死叉的票 |
| `EXCLUDE_BELOW_YELLOW` | True | 排除价格跌破黄线的票 |
| `MIN_SCORE` | 25 | 最低入围总分 |
| `TOP_N` | 30 | 最终输出前 N 名 |
| `REQUEST_DELAY` | 0.15s | 每只股票间的请求间隔 |
| `MIN_LISTING_DAYS` | 120 | 最少需要的历史 K 线条数 |

> 将 `EXCLUDE_BELOW_YELLOW` 设为 `False` 可放宽筛选，纳入更多低位潜在地板票（同时也增加了误入率）。

## 与现有项目的关系

选股模块完全独立新增，**不修改任何原有文件**：

- `criteria/ruthless.py` → 只读 import `脚本/backtest_ruthless.py`（复用 DEMA/KDJ/砖柱/B1/B砖 等函数）
- `criteria/wave.py` → 只读 import `脚本/fetch_wave_data.py`（复用 MACD/Fib/背离/摆动点 等函数）
- `scanner.py` → 只读 import `脚本/fetch_wave_data.py`（复用 fetch_kline 抓取数据）
- 输出 → 喂给 `观察仓/` 目录，人工审核后加入 watchlist（与现有工作流衔接）
